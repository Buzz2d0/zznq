我尝试用 golang 编写一些核心代码，再利用 cgo 的特性将其打包成 dylib 文件，以便于我在objc 中调用（因为我 objc 写的很烂）。

先写个很简单的 cgo 程序 main.go ：

package main

//void HelloWorld();
import "C"
import "fmt"

//export HelloWorld
func HelloWorld() {
	fmt.Println("Hello World!!!")
}

func main() {}

代码中有几个地方需要注意下：

  • import "C" 语句则表示开启了CGO特性
  • //export HelloWorld CGO 指令将其导出为C语言函数

编译为dylib动态库： $ go build -buildmode=c-shared--ldflags="-w -s" -o cgodemo.dylib cgoDemo/main.go

查看下文件信息：

$ file cgodemo.dylib
cgodemo.dylib: Mach-O 64-bit dynamically linked shared library x86_64
$ otool -L cgodemo.dylib
cgodemo.dylib:
        cgodemo.dylib (compatibility version 0.0.0, current version 0.0.0)
        /usr/lib/libSystem.B.dylib (compatibility version 1.0.0, current version 1292.100.5)

调用下动态库测试下 main.c ：

#include <stdio.h>
#include <dlfcn.h>

int main()
{
    char *so_path = "./cgodemo.dylib";
    void *lib = dlopen(so_path, RTLD_LAZY);
    void (*HelloWorld)();
    HelloWorld = dlsym(lib, "HelloWorld");
    HelloWorld();
}

测试：

$ clang main.c -o main
$ ./main              
Hello World!!!

amazing!

在 objc 下尝试加载动态库：

#import <Foundation/Foundation.h>
#include "dlfcn.h"

int main(int argc, const char * argv[]) {
    @autoreleasepool {
        void *lib = dlopen("../cgodemo.dylib", RTLD_LAZY);
        if (NULL != lib){
            NSLog(@"not found dylib file...");
            return 1;
        }
        void (*HelloWorld)(void) = dlsym(lib, "HelloWorld");
        if (NULL != lib){
            HelloWorld();
        }else{
            NSLog(@"not found HelloWorld()");
        }
        dlclose(lib);
    }
    return 0;
}

然而并非一番风顺，打印 not found HelloWorld()，找不到 HelloWorld 函数，google 了好久也没排查出原因，路过的您如果知道咋解决麻烦🙏告诉我一下😁。

放弃？是不可能的，这辈子都不可能

排查问题时发现了 mobile 工具：https://github.com/golang/mobile ，可以在 Java & Objc 中调用 golang, 相关文档在 https://github.com/golang/go/wiki/Mobile 。

虽然说了 gobuild 并不支持单独使用，但是 $ gomobile bind -target=ios golang.org/x/mobile/example/bind/hello 可以将 hello package 生成 Hello.framework, 然后就可以在 objc 中直接调用了 🐂🍺

所以首先解决下如何修改参数让其可以编译为 macos 平台的 Framework，google 到一个 https://github.com/golang/go/issues/31258 ，他是这么解决的：

// cmd/gomobile/env.go
...
case "386", "amd64":
			clang, cflags, err = envClang("iphonesimulator")
			cflags += " -mios-simulator-version-min=" + buildIOSVersion
// 		修该为：
			clang, cflags, err = envClang("macosx")
			cflags += " -mmacosx-version-min=10.11"
...

我 fork 下来修改并提交了 https://github.com/Buzz2d0/mobile ，在测试发现这里版本-mmacosx-version-min=10.11 可以根据自己系统版本来决定，我用的 bigsur 11.3 就改为了 10.15 。

安装修改后的 gomobile

$ git clone https://github.com/Buzz2d0/mobile 
$ cd mobile 
$ go build  golang.org/x/mobile/cmd/gomobile
$ gomobile init

敲个测试代码 lib/lib.go：

package lib

import "fmt"

func Greetings(name string) string {
	return fmt.Sprintf("Hello, %s!", name)
}

创建 Lib.framework ： $ ../mobile/gomobile bind -target=ios demo/lib

把 Lib.framework 导入到 Xcode 中，具体操作：https://github.com/golang/go/wiki/Mobile#building-and-deploying-to-ios-1

main.m :

#import <Foundation/Foundation.h>
#import <objc/runtime.h>
#import "Lib/Lib.h"

int main(int argc, const char * argv[]) {
    @autoreleasepool {
        NSString * str = LibGreetings(@"this is a golang function");
        NSLog(@"%@", str);
    }
    return 0;
}

编译运行 成功啦！
